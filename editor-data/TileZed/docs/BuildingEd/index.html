Config Files

Settings for things like furniture, tiles, and templates are saved in text files in the .TileZed directory in the user's home directory.  On Windows, this will usually be C:\Users\USERNAME\.TileZed.  When a new version of BuildingEd is run for the first time, these config files will be updated as needed.

BuildingFurniture.txt

The BuildingFurniture.txt file contains all the furniture groups defined in the Tiles Dialog.

BuildingTemplates.txt

The BuildingTemplates.txt file contains all the building templates defined in the Templates Dialog.

BuildingTiles.txt

The BuildingTiles.txt file contains all the non-furniture tile categories defined in the Tiles Dialog.

Tilesets.txt

The Tilesets.txt file contains information about tilesets.  The list of tilesets can be edited in the Tilesets Dialog. The names of the tileset image files are saved relative to the Tiles directory specified in the Tilesets Dialog.  These tilesets are added to any .tmx files (i.e., TileZed map files) exported by BuildingEd.

TMXConfig.txt

The TMXConfig.txt file contains a list layer names that will be added to exported .tmx files.  Tile layers are defined with "tile = NAME" and object layers with "object = NAME".  If "NAME" starts with a level prefix such as "0_NAME", the layer will only be added to the specified level, otherwise the layer will be added to every level.

Autosaving

To avoid losing too much work in the event the editor crashes, every 2.5 minutes BuildingEd will save the current building if it has unsaved changes.  The name of the auto-saved file will end with the ".autosave" extension.
  • If the building is new and has never been saved, the autosave file will be in the .TileZed directory in the user's home directory.  On Windows, this will usually be C:\Users\USERNAME\.TileZed.  The name of the file will be "untitled.autosave".
  • If the building was loaded from an existing .tbx file, the name of the autosave file will be the same as the .tbx file plus ".autosave" in the same directory as the .tbx file.  For example, if the current building was loaded from "C:\MyBuildings\MyCoolBuilding.tbx", the autosave file will be named "C:\MyBuildings\MyCoolBuilding.tbx.autosave".
The autosave file will be deleted if the building is saved by the user, if a new building is created/opened, or if BuildingEd is closed.
