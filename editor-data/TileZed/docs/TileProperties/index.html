Tile Properties Editor

The Tile Properties Editor is used to edit .tiles files. A .tiles file is a binary file the game uses to determine which tiles are wall tiles, door tiles, etc.  A .tiles file stores a list of tilesets and the properties assigned to each tile in a tileset.  Every tile property is just a string name plus string value.  The editor supports Boolean, Integer, String and Enum (enumeration) properties.

TileProperties.txt

The names and types of all the properties that can be edited are specified in the TileProperties.txt file.  You can easily add new property-editing controls to the editor by editing the TileProperties.txt file in a text editor program.  The TileProperties.txt file is read in every time the Tile Properites Editor is displayed.  So you may close the editor (but keep TileZed running), edit TileProperties.txt, then open the editor again to see the changes.

There is also a set of simple commands that can modify the properties in a .tiles file when it is read in.

Default Values

All properties in the editor have a default value.  When a property is set to its default value, no property is written to the .tiles file.  The label for an editor control (like a checkbox) is displayed in bold when a property does not have its default value.

Boolean Properties

property
{
    Type = Boolean
    Name = CollideNorth
    ShortName = collideN
    Default = false
    ReverseLogic = false
}

Boolean properties are displayed as a checkbox in the editor.  In a .tiles file, a Boolean property is stored as ShortName="".
  • Name is displayed as the label beside the checkbox in the editor.
  • ShortName is the name of the property written to the .tiles file.
  • Default is the default value for the property.  It may be true or false.
  • ReverseLogic causes the property to be written to the .tiles file when the property does have its default value.  This is the opposite of the usual behavior.

Integer Properties

property
{
    Type = Integer
    Name = WaterAmount
    ShortName = waterAmount
    Min = 0
    Max = 1000000
    Default = 0
}

Integer properties are displayed as a spinbox in the editor.  In a .tiles file, an Integer property is stored as ShortName=N where N is the integer value of the property.
  • Name is displayed as the label beside the spinbox in the editor.
  • ShortName is the name of the property written to the .tiles file.
  • Min and Max limit the values the user can enter in the spinbox.  Min must be less than Max.
  • Default is the default value for the property.  It must be between Min and Max inclusive.

String Properties

property
{
    Type = String
    Name = ContainerType
    ShortName = container
    Default =
}

String properties are displayed as an edit field with a drop-down list in the editor.  The drop-down list is populated with all the values of the property found in the .tiles file.  In a .tiles file, a String property is stored as ShortName=S where S is the string value of the property.
  • Name is displayed as the label beside the edit field in the editor.
  • ShortName is the name of the property written to the .tiles file.
  • Default is the default value for the property.  It can be any one-line string.

Enum Properties

property
{
    Type = Enum
    Name = StairStyle
    ShortName =
    Enums
    {
        None =
        BottomW = stairsBW
        MiddleW = stairsMW
        TopW = stairsTW
        BottomN = stairsBN
        MiddleN = stairsMN
        TopN = stairsTN
    }
    Default = None
    ValueAsPropertyName = true
    ExtraPropertyIfSet =
}

Enum properties are displayed as a combobox in the editor.  In a .tiles file, an Enum property may be stored as either ShortName=E  or as E="", where E is one of the possible values of this property.
  • Name is displayed as the label beside the combobox field in the editor.
  • ShortName is the name of the property written to the .tiles file, but only when ValueAsPropertyName is false.
  • Enums lists all the possible values of this property.  For each A=B listed, A is the name displayed in the editor combobox, and B is the name written to the .tiles file.  If B is left empty, then A is written to the .tiles file.
  • Default is the default value for this property.  It must be one of the values listed in the Enums block.
  • ValueAsPropertyName controls how the property is written to the .tiles file.  If you want SomeEnum=SomeValue, set this to false.  If you want SomeValue="", set this to true.  In the example above, if BottomW is chosen in the editor, then stairsBW="" will be written to the .tiles file.
  • ExtraPropertyIfSet specifies the name of an additional property that will be set on a tile if ValueAsPropertyName=true and the value of this property is not the default.  This is used to set the wall="" property on a tile if the WallStyle property is also set on that tile.
